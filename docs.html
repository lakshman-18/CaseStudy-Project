Scheduling Algorithms

What is CPU Scheduling?

CPU Scheduling is a process of determining which process will own CPU for execution while another process is on hold. The main task of CPU scheduling is to make sure that whenever the CPU remains idle, the OS at least select one of the processes available in the ready queue for execution. The selection process will be carried out by the CPU scheduler. It selects one of the processes in memory that are ready for execution.

GHOST BSD Operating System

GhostBSD is a desktop operating system based on FreeBSD. It uses a modified version of the FreeBSD kernel and includes a variety of open-source software packages. GhostBSD uses the ULE scheduler, which is a priority-based scheduler that supports deadline scheduling. In ULE scheduler, each process is assigned a priority value based on its dynamic behavior. The priority value ranges from 0 to 255, where 0 is the highest priority and 255 is the lowest priority. The ULE scheduler uses priority values to decide which process to run next

WHY Deadline Scheduling

In addition to priority values, the ULE scheduler also supports deadline scheduling. In deadline scheduling, each process is assigned a deadline by which it must complete its execution. If a process misses its deadline, it is penalized by lowering its priority value. This ensures that processes that are close to missing their deadlines are given higher priority than processes that have more time to complete.

GhostBSD uses the ULE scheduler with deadline scheduling to ensure that critical tasks are completed on time. For example, if a process is responsible for handling user input, it may be assigned a deadline of a few milliseconds to ensure that it responds quickly to user actions. Important CPU scheduling Terminologies

  • Burst Time/Execution Time: It is a time required by the process to complete execution. It is also called running time.
  • Arrival Time: when a process enters in a ready state
  • Finish Time: when process complete and exit from a system
  • Multiprogramming: A number of programs which can be present in memory at the same time.
  • Jobs: It is a type of program without any kind of user interaction.
  • User: It is a kind of program having user interaction.
  • Process: It is the reference that is used for both job and user.
  • CPU/IO burst cycle: Characterizes process execution, which alternates between CPU and I/O activity. CPU times are shorter than the time of I/O.
